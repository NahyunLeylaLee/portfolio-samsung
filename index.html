Semiconductor Data Pipeline

Apache NiFi Based Real-Time Data Processing System

Samsung SDS Project

Project Overview

This is the first semiconductor data pipeline I personally designed, built, and operated at Samsung SDS using Apache NiFi. Due to security considerations, some detailed information in the images has been blurred.

This was my first project utilizing Apache NiFi, with custom processors developed using Java and Spring Boot, thoroughly tested with JUnit.

Apache NiFi Java Spring Boot JUnit Oracle DB FTP/SFTP

Key Improvements Over Traditional Java Pipeline

  • Enhanced visualization enables immediate identification of problem areas
  • Improved operational efficiency through real-time monitoring
  • Modular design with processor groups for better organization
  • Streamlined debugging and maintenance processes
01

Pipeline Architecture

Overall system design and processor group organization

The pipeline architecture demonstrates the power of Apache NiFi's visual design. Multiple processors are organized into logical groups, making it easy to identify bottlenecks and issues at a glance.

Design Philosophy: Modular processor groups enable efficient troubleshooting and system monitoring, significantly improving operational efficiency compared to traditional Java-based pipelines.
02

Job Scheduler

Automated data collection scheduling and filtering

The Job Scheduler component manages time-based data collection from semiconductor facilities through FTP and SFTP protocols. It operates at specified intervals to retrieve newly generated data from production equipment.

Smart Filtering: Files are filtered according to Samsung Electronics' requirements before collection, ensuring only relevant data enters the pipeline.
Protocol Support: Seamless integration with both FTP and SFTP for secure and flexible data transfer from various facility systems.
03

Collector

Duplicate detection and data acquisition

The Collector performs intelligent duplicate detection by checking collected filenames against the database before fetching new data. This prevents redundant processing and ensures data integrity.

Duplicate Prevention: Database verification ensures each file is collected only once, maintaining data consistency.
Automated Tracking: Newly collected file metadata is automatically stored in the database for future reference and auditing.
Efficient Retrieval: Uses FetchFTP and FetchSFTP processors to retrieve only new data from semiconductor facilities.
04

Parser

Data transformation and extraction

The Parser component retrieves filenames from the database and extracts the specific data required by Samsung Electronics from the collected files.

Custom Parsing Logic: Tailored to Samsung Electronics' data requirements for precise extraction.
Database Integration: Fetches file metadata from tracking tables to process the correct datasets.
05

Transfer

Database persistence and error handling

The Transfer component is responsible for persisting parsed data into the Oracle database with comprehensive error handling mechanisms.

Oracle DB Integration: Reliable storage of processed semiconductor data.
Error Logging: Failed insertions are captured and logged in a dedicated error table for troubleshooting and recovery.
